If your **Expo Android build fails** with OneSignal and shows errors like:

- `Module was compiled with an incompatible version of Kotlin`
- `Kotlin metadata 2.2.0 expected 2.0.0`
- `Task :app:compileDebugKotlin FAILED`

This guide shows the **exact cause and the working fix**.

---

## The Exact Problem

When running `npx expo run:android`, the build fails at the `:app:compileDebugKotlin` step.

### Common Error Messages

```text
Module was compiled with an incompatible version of Kotlin
Internal compiler error: source must not be null
Kotlin metadata 2.2.0 expected 2.0.0
```

---

## Why This Happens

This happens because of a **Kotlin version mismatch** between the library and the project environment:

* **Expo SDK 53** defaults to **Kotlin 2.0.x**.
* Newer **OneSignal Android SDK** versions (5.7.x+) are compiled with **Kotlin 2.2.x**.
* Kotlin cannot compile libraries built with a newer metadata version, leading to a build failure.

This is a classic **version compatibility issue** between the SDKs.

---

## Working Fix: Pinning OneSignal Version

Instead of upgrading Kotlin (which Expo SDK 53 doesn't fully support yet), the most stable solution is to **pin OneSignal to a compatible version (5.6.0)**.

### Step 1: Update Gradle Properties

Open `android/gradle.properties` and add the following line:

```properties
android.onesignalVersion=5.6.0
```

### Step 2: Force Version in Build Configuration

Open `android/build.gradle` and add this logic at the bottom of the file:

```gradle
ext {
  oneSignalVersion = findProperty("android.onesignalVersion") ?: "5.6.0"
}

subprojects {
  project.configurations.all {
    resolutionStrategy.force "com.onesignal:OneSignal:${oneSignalVersion}"
  }
}
```

### Step 3: Clean and Rebuild

Run the following commands to clear the cache and start a fresh build:

```bash
cd android
./gradlew clean
cd ..
npx expo run:android
```

---

## Result and Summary

After applying these changes:
- ✅ Android build completes successfully.
- ✅ OneSignal functionality remains intact.
- ✅ No more Kotlin metadata version errors.

### Compatibility Table

| Tool            | Kotlin Version | Status |
| --------------- | -------------- | ------ |
| Expo SDK 53     | Kotlin 2.0.x   | Base   |
| OneSignal 5.7.x | Kotlin 2.2.x   | ❌ Incompatible |
| OneSignal 5.6.0 | Kotlin 2.0.x   | ✅ Compatible |

> [!TIP]
> This issue will likely be resolved in future Expo SDK versions when Kotlin versions are updated. Until then, pinning OneSignal is the recommended approach.
